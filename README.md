# System Troubleshooting Playbook

This repository contains notes, playbooks and investigation strategies for diagnosing issues in distributed systems.

The goal is to document practical approaches for:

- API troubleshooting
- Integration debugging
- Incident investigation
- System observability

These notes are based on real-world support and platform investigation scenarios.

## Topics covered

HTTP fundamentals  
API latency investigation  
Error 500 troubleshooting  
Integration failures  
Log analysis  
Monitoring strategies

## Structure

/http → protocol fundamentals and debugging  
/apis → API troubleshooting  
/integrations → integration diagnostics  
/observability → logs and monitoring  
/case-studies → practical investigation scenarios
